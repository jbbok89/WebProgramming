sudo 원데이 분석 (CVE-2019-14287)
CVE | cucu | 2026-04-07
Sudo 취약점(CVE-2019-14287)은 -u 옵션의 입력값 검증 미흡을 이용하여 /etc/sudoers 파일의 보안 정책을 우회하고 root 권한을 획득하는 공격입니다.
공격자는 특정 사용자 권한으로 명령을 실행하는 -u 옵션에 UID 값으로 -1 또는 4294967295(UINT_MAX)를 입력합니다.
시스템은 이 값을 0(root)이 아닌 것으로 판단하여 권한 제한 정책을 통과시키지만, 실제 실행 단계에서는 이를 '권한 변경 없음'으로 해석하여 sudo의 기본 권한인 root(0)로 명령을 실행하게 됩니다.
방어 방법 - Sudo 버전 업데이트: 취약점이 해결된 1.8.28 이후 버전으로 업데이트를 수행합니다.
- 입력값 유효성 검증 강화: lib/util/strtoid.c와 같은 소스 코드 레벨에서 UID 입력값으로 -1, UINT_MAX 또는 유효하지 않은 구분자가 사용될 경우 에러 처리가 되도록 패치를 적용합니다.
- 보안 정책 및 권한 검사 함수 최신화: sudo_strtoid()와 같이 입력값을 숫자로 변환하고 검증하는 함수가 정상적으로 작동하는지 확인하고, 정책 확인 시 반환값을 엄격히 체크합니다.
