PIE (Position-Independent Executable)
Security | cucu | 2026-04-12
PIE(Position-Independent Executable)는 실행 파일의 코드와 데이터 영역까지 메모리상의 무작위 주소에 매핑하여 보안을 강화하는 기술입니다.
기본적인 ASLR이 스택, 힙, 라이브러리 주소만 무작위화하는 것과 달리, PIE를 적용하면 실행 파일(코드 영역) 자체의 위치도 실행 시마다 바뀝니다.
주요 개념 - PIC(Position-Independent Code) 메모리의 어느 위치에 적재되어도 코드의 의미가 훼손되지 않는 코드를 말합니다. rip 레지스터를 기준으로 데이터를 상대 참조하므로, 전체 주소가 바뀌어도 내부 간격이 일정하여 정상 실행이 가능합니다.
- 시스템 제약: ASLR이 적용되지 않은 시스템에서는 PIE가 적용된 파일이라도 무작위 주소에 적재되지 않을 수 있습니다.
