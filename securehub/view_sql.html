[Injection] SQL 인젝션 분석 및 방어

VUL | moi_Hack | 2026-04-09

1. 공격 원리

웹사이트의 로그인 창이나 검색창에 아이디를 입력하면, 서버 내부에서는 다음과 같은 SQL 쿼리가 실행됩니다. 

SELECT * FROM users WHERE id = '입력값' AND pw = '입력값';

이때 공격자가 아이디 칸에 ' OR '1'='1 이라는 값을 넣으면 쿼리는 다음과 같이 변합니다. 

SELECT * FROM users WHERE id = '' OR '1'='1' -- ' AND pw = '...';

2. 주요 피해 유형

  • 데이터 유출: 사용자 개인정보를 통째로 탈취
  • 데이터 조작: 관리자 비밀번호 변경 등 권한 남용
  • 시스템 장악: DB 명령어를 통한 OS 접근

3. 방어 방법

가장 확실한 방어법은 입력값을 명령어가 아닌 단순 문자열로 처리하는 것입니다.

  • Prepared Statement 사용 (가장 권장됨)
  • 입력값 검증 및 필터링
  • 에러 메시지 노출 제한
