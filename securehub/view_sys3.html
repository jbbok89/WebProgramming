ROP(Return Oriented Programming)
VUL | cucu | 2026-04-11
ROP(Return Oriented Programming)는 프로그램 내부에 이미 존재하는 코드 조각들인 가젯(Gadget)을 조합하여 공격자가 원하는 실행 흐름을 만들어내는 공격 기법입니다.
공격자는 실행 권한이 있는 메모리 영역(바이너리 또는 라이브러리)에서 끝이 ret으로 끝나는 리턴 가젯들을 찾아내고, 이들의 주소로 스택을 구성하여 연쇄적으로 실행함으 로써 보안 방어 체계를 우회합니다.
핵심 개념 - 리턴 가젯(Return Gadget): ret 명령어로 끝나는 코드 조각입니다. ret은 다음 실행할 주소를 rip에 넣고 이동하므로, 이를 이용해 여러 가젯을 이어 붙일 수 있습니다.
- PLT / GOT: 동적 링크 방식에서 함수 호출을 위해 사용됩니다. PLT는 함수의 실제 주소를 찾는 코드 영역이고, GOT는 실제 주소가 저장되는 데이터 영역입니다. 공격자는 이를 참조하거나 변조하여 원하는 함수를 실행합니다.
- 라이브러리 OFFSET: 동일한 라이브러리 파일 내에서 함수 간의 거리는 항상 일정합 니다. 따라서 메모리상의 주소 하나만 알아내면 다른 모든 함수의 주소를 계산할 수 있습니다.
환경별 차이 및 주의사항 - x64 vs x86 64비트 환경은 인자를 레지스터에 넣고 16바이트 스택 정렬이 필수적 입니다. (정렬되지 않으면 movaps 등에서 오류 발생) 반면 32비트(x86)는 인자를 스택에 쌓으며 별도의 정렬 규칙이 없습니다.
- 동적 vs 정적 링크 동적 링크 프로그램은 PLT/GOT가 반드시 필요하지만, 라이브러리 코드가 통째로 포함된 정적 링크 프로그램은 가젯과 시스템 콜을 직접 호출하는 방식을 사용합니다.
ROP 대응 방안 - PIE(Position Independent Executable) 적용 바이너리가 실행될 때마다 주소를 무작위화하여 가젯의 위치를 찾기 어렵게 만듭니다. - CFI(Control Flow Integrity) 프로그램의 정상적인 실행 흐름을 미리 정의하고, 이를 벗어나는 비정상적인 점프가 발생하면 프로그램을 종료합니다. - Shadow Stack: 리턴 주소만을 저장하는 별도의 안전한 스택을 운영하여, 스택의 리턴 주소가 변조되었는지 검증합니다.
